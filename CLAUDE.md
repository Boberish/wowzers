# Project Rift — Godot 4 Port

A co-op roguelike built on **MMO trinity combat with movement removed**, replaced by **timed active decisions**. Each role isolates one raiding verb (mitigate / drive-rhythm / interrupt / keep-alive). Solo-playable with AI allies; dream is 4-player co-op. Data-driven boss encounters. Hades-style deckbuilder progression (pick 1 of 2 Aspects per class at run start).

The four browser prototypes in `poc/` are the **executable design spec**. Port the *behavior*, not the JS. Lift the tuned numbers as starting values, then rebalance in-engine.

## Repo layout & how to run
- `godot/` — the Godot 4.7 project (`res://`). GDScript, GL Compatibility renderer. See `godot/README.md`.
- Godot is installed at `~/.local/bin/godot` (4.7-stable, standard/non-mono build).
- Headless balance sim: `godot --headless --path godot --script res://sim/sim_runner.gd -- --seeds=400`
- Open the editor (WSLg GUI): `godot --path godot --editor` (add `--rendering-driver opengl3` if the window fails).
- **On a fresh checkout, first run `godot --headless --path godot --import`** to build the global-class cache, or `--script` runs can't resolve `class_name`s.

A press is graded against the next answerable beat by press-to-impact delta (`TuningConfig.strike_perfect/good/graze` = 0.14/0.34/0.50s): PERFECT/GOOD fully negate a DODGEABLE beat, GRAZE takes `graze_mult` 0.5, MISS eats it; **BLOCKABLE** beats land 0.25/0.5/0.75 even when answered (the partial-block fantasy). Presses have `dodge_recovery` 0.35s; a press that answers nothing — or dodges a FEINT beat (= **BAITED**) — burns `dodge_whiff_cd` 1.3s, so panic-mashing eats the rest of the combo. ## Locked decisions (made 2026-07-01)
- **Engine/language:** Godot 4.x, **GDScript** (best web-export support + iteration speed; C# rejected).
- **Netcode:** **server-authoritative**, one **dedicated headless Godot server** over **WebSocket** (works browser + native).
- **Server hosting:** self-hosted on the dev's desktop in a **Docker container**, exposed via a **TLS tunnel** (Cloudflare Tunnel → gives the `wss://` browsers require). $0/mo; same container lifts to a VPS unchanged later.
- **Clients:** **browser (WASM) export** first (send a link, no install); native desktop later for free.
- **Balance sims:** run **in-engine, headless** (`godot --headless`) so the *same* code produces game + sim results (single source of truth). Analysis/plots can live in Python/pandas over the exported CSVs.
- **AI:** boss = data-driven encounters; allies = per-role `policy.act(obs)` sharing the human input surface (human & AI seats interchangeable). Skill = policy parameter (latency/accuracy).

## The one non-negotiable architectural rule
The combat engine (`CombatCore`) is a **pure, deterministic, DOM-free / Node-free module**. The *same* `update(state, dt)` runs in three places: the live client, the authoritative server, and headless batch sims. That is only possible if it has **zero rendering, zero wall-clock time, zero unseeded randomness, zero mutable module-level globals**. Everything else is a client of it.

### Determinism rules (build in from day one — never retrofit)
1. **Fixed 30 Hz timestep.** No `requestAnimationFrame`/wall-clock dt. Accumulator drains `update(1.0/30.0)`. `render()` reads state at display rate.
2. **Integer tick is truth.** `state.tick += 1`; derive `time = tick * DT`. Store every absolute-time field (cooldowns, dodge windows, DoT expiries, enrage) as ticks.
3. **One seeded PRNG in state** (mulberry32), advanced only inside `update()` in a fixed, documented order. Cosmetic randomness (floating combat text) uses a *separate* client-only RNG that never touches authoritative state.
4. **Input is a tick-stamped queue** drained at the top of the owning tick — never mutate state inside an input event.
5. **No hard-coded balance literals.** All tuning lives in an overridable `TuningConfig` Resource. **No mutable module-level globals** (the prototypes mutate `DEF/MOM_MAX/INT_CD/SIL_DUR` — those must live on per-seat/per-run state or they leak across players & parallel sims).

## Seat model (makes human = AI = sim agent)
Every participant is a **seat**: `(role, observation-adapter, action-set, policy, fidelity)`.
- `perform(action)` is the single input surface. Keyboard/mouse is a thin adapter that emits `perform()`. AI allies emit the same via `policy.act(obs)`. Sim agents are identical, headless.
- **Two fidelities:** STAT-BLOCK (`ally.dps * f(hp%)` — cheap, for big sweeps) and FULL (policy drives the real ability engine — playable co-op AI + high-accuracy sims). Validate stat-block against occasional full runs.
- **Win condition = the damage line:** `boss.hp -= Σ (living ally.dps * f(hp%)) * dt`. Default `f = 0.3 + 0.7*hp%`. Brinkwarden overrides `f` (bloodied allies hit harder) — keep as a hook, not a special-case.
