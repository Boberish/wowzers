# research/ — the game-design knowledge base

**What this is.** Reference notes on how other games design classes, specs, builds, and
cards/abilities — researched from wikis + model knowledge, stored here so ANY future design pass
(deck rebuilds, new specs, boss work) can read them instead of re-researching. Born 2026-07-09 for
the Twinfang·Tempo branch-slate pass (see `TEMPO-PLAN.md`).

**How to use these.** They are idea quarries, not specs. Everything borrowed must pass the
project's filter — **"borrow the grammar, innovate the sentence"** (`DECK-LAYOUT.md` §6): a stolen
system has to do something only our timing-combat / deterministic-AI engine could, or it's a
repack. Each file ends with STEAL CANDIDATES already mapped to our grammar (branches · dial-lanes ·
EASE dial · rig WHEN→THEN · keystones · signature CD).

## The uniform template (every file follows it)

1. **Build grammar** — how the game structures specs/classes/builds, one paragraph.
2. **Spec/class/aspect roster** — name · fantasy · core mechanic · signature moment.
3. **Card/ability/upgrade design patterns** — rarity, pools, synergy grammar, build-arounds vs
   bread, greed/risk mechanics, what makes their best cards feel great.
4. **Difficulty & self-tuning systems** — player-set difficulty, ascension-style ladders, affixes.
5. **Party/co-op interplay** — where the game has it.
6. **STEAL CANDIDATES for Rift** — mapped to our grammar, filter applied; includes a
   Twinfang/rogue-flavored subsection (dual-blade / rhythm / timing-melee angles).
7. **Sources** — URLs consulted.

Notes are point-in-time (researched 2026-07-09); live games move — verify numbers before leaning
on them for balance.
